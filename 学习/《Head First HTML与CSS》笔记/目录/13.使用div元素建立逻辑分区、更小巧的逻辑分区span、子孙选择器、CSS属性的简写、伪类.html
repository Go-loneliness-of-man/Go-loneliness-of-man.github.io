使用div元素建立逻辑分区、更小巧的逻辑分区:span、子孙选择器、CSS简写、伪类

一、使用div元素为页面划分逻辑分区

虽然有了id属性和class属性支持的“类”，让我们可以较为容易的设计元素的样式，但为了更加随心所欲的设计、更改元素的样式，引入了“逻辑分区”这个概 念，逻辑分区就是页面上彼此相关的紧挨的一组元素，引入它是为了使页面看起来更加结构化，使页面更加易于设计，识别。
划分逻辑分区要依靠div元素，你可以将一些挨着的元素放在一个div元素里，这样你就为这些元素提供了一个父元素，而它的表现就是一个元素，你完全可以 将它当作一个大的元素来使用。具体用法就是用它包围你想要归到一起的元素并且使用id属性为其命名，像这样:
<div id=wangchao>一些相关的元素</div>
而为其设计样式时也正是如同被id属性标记了的元素一样:
#wangchao{ 属性:样式; }
可以说，它就是一个元素，另外，div元素也可以嵌套使用。

二、更小巧的逻辑分区:span

div元素其实是用来为大的块元素创建逻辑分区的，它本身就是一个块元素，会带来换行，而一些内联元素或一段文本使用div元素未免有些不合适，这就可以 用它的内联版:span元素，它可以将任意内容包围起来形成一个元素，以便于指定样式，不过为了与div元素相区别，它使用class属性命名，相应的，在使用 时它的书写也如同类:
元素名.class名{ 属性:样式; }

三、子孙选择器

子孙选择器可以用来选择某一个元素的一个嵌套元素，这样就能特别的指定一个元素的样式，虽然使用id属性也可以达到同样的目的，但相比之下子孙选择器更 加简便，具体使用很简单，在父元素与嵌套元素之间空开一个空格就可以:
父元素 嵌套元素{ 属性:样式; }
这个嵌套并不是只能有两层，它是可以无限延伸下去的。
特别的，如果父元素已经有了自己的id，那也可以这样来写:
#父元素id 嵌套元素{ 属性:样式; }

四、CSS属性的简写

很多CSS属性都是可以合并在一起的，这称为简写，例如background-color、background-image、background-position、background-repeat等 等，都可以直接写成background，然后在后边以任意顺序添加属性值即可，当然是要用空格空开的。类似的还有font系列属性等，都可以这样做。

五、伪类

伪类在使用上很像类，但不同于类的是它不需要使用class属性定义名字，因为它是事先和浏览器约定好的，有固定名称的。使用它最多的是地方就是a元素，因 为链接的样式总是随着状态的改变而改变，例如未访问的链接通常是蓝色，已访问过的链接通常变为紫色，那就是因为伪类在起作用，下面是一个使用伪类为a元 素写的规则:
a:link{ color:blue; }
a:visited{ color:purple; }
a:hover{ color:white; }
link、visited、hover分别代表的是未访问、已访问、悬停(即鼠标悬停在链接上)，浏览器会根据元素此时的状态来实时的将其分配到对应的类中，这样元素 的样式自然也相应的改变。不止上述这三种，伪类还有很多种,用于对各种具有某些特征的元素进行归类。
