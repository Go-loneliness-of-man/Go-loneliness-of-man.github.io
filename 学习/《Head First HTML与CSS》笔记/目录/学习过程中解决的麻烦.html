用编译器编写的HTML中的汉字在浏览器里显示乱码: 原因是HTML的编码方式与浏览器默认的编码方式不同，才出现乱码。 例如火狐浏览器采用utf-8，当其显示乱码时，只要将HTML也设置为utf-8形式即可，具体解决办法就是在head中添加这段代码即可:
<meta charset="utf-8">
消除元素间压缩的空格 浏览器会将源代码中元素间的所有换行符和空格压缩为一个空格，导致页面的元素间总有一个空格存在，解决办法:
空格和换行符也属于字符，那么只需要将父元素的字体大小font-size设置为0，那么空格和换行符也就不再占据空间，至于子元素继承的问 题，只需要再为子元素分别设置字体大小，即可恢复子元素样式。
块元素和内联元素的内、外边距问题 块元素的水平外边距不会合并，但在垂直方向上块元素间的边距会合并，合并即重叠，以边距大的一方为准;内联元素外边距只在水平方向上 起作用，不会合并。
块元素内边距没有异常;内联元素的内边距有一些不同，它的内边距会正常撑开自己的盒子，由于自身变化也会在水平方向上正常将其它元素 推开，但在垂直方向上不会，在垂直方向上它会遮挡住其它元素。
当两元素嵌套时:
当嵌套时且父元素与子元素间没有任何内容时，子元素的外边距在垂直方向上会与父元素的外边距合并，造成在垂直方向上，子元素不能在父 元素中移动，解决方法就是设法在其间加一些内容。至于"为什么"涉及到CSS的工作原理。
