一、字符实体

由于html中<和>具有指明标记的作用，因此无法直接通过在html中打出这两个标记来显示，其实要想在浏览器中显示像这类的特殊字符，可以通过使用 字符实体来实现。
就像摩尔电码一样，字符实体就是对特殊字符进行编码，使用一些其它字符来代替这些特殊字符，当浏览器读到字符实体时会自动将其翻译出来，例如< 对应的字符实体就是& lt，而>对应的就是& gt。
所有字符实体的开头都使用&,当你单独使用它的时候，就得输入它所对应的字符实体“& amp;”。

二、将网站发布到Web上

将网站发布到Web上只需要完成三件事即可:

  1. 找一家托管公司
  2. 为你的网站选一个域名
  3. 将网站上传到托管公司的服务器上
要在Web上发布网页，就需要一个在Web上全天候工作的服务器，最好的选择是找一个托管公司。让托管公司负责保证服务器一直运行的诸多细节性问题，当然 这需要一些经费，但不贵，一般大概几十块钱一个月。
域名就是一个在Web上独一无二的名字，可以用来定位你的网站。举个栗子说明，在“www.wangchao.com”中，www代表服务器的名字，wangchao.com就是域 名，其中结尾的.com中包含有一些信息，比如.edu代表教育网域名。为了避免域名重复，域名由一个机构集中控制，以确保不会多个人使用同一个域名，也因此， 要保留你的域名，每年需要缴纳注册费。至于注册域名的事，大部分托管公司都会提供域名注册的服务，事实上交给它们办是最容易的。
网站名不是域名，网站名是服务器名加域名，在一个域名上可以创建多个网站，只要确保它们的服务器名不同即可，所以域名就像一块地，不同的服务器名会在这 块地上建立不同的网站。
解决了托管公司和域名的问题，就只剩将网站传到服务器上了，就是将网站文件从你自己的根文件夹转移到Web服务器上的根文件夹，根文件夹即顶级文件夹。传 输的方式有很多种，大多数公司都支持一种叫做FTP（文件传输协议）的文件传输方法，当然现在也可以使用云来传输。

三、URL——统一资源定位符

你在浏览器输入的Web地址是一个全局地址，被称为URL（统一资源定位符），可以用来定位Web上的任意资源。现在来看看一个URL:
http://www.wangchao.com/index.html
它由三部分组成:获取资源的协议HTTP、网站名、从根文件夹到资源的绝对路径。协议部分告诉浏览器应当使用什么方法来获取资源，网站名高速浏览器要从互联网 上的哪个计算机获取资源，最后绝对路径告诉服务器如何找到资源。一个典型的URL由一个协议、一个网站名、一个资源的绝对地址组成。
Web页面的地址都以http开头，HTTP是超文本传输协议，就是在Web上传输超文本文档（其实是Web页面需要的任何文件）公认的一种方法。要定位Web上的某一个 资源，只需要知道它所在的服务器以及资源的绝对路径，就可以创建一个URL,浏览器会使用某个协议来获取这个资源，大多数情况下是HTTP，言外之意也就是说URL 还可以指定用来获取资源的协议。
HTTP也可以说是一个简单的请求和响应协议，每次在浏览器的地址栏中输入一个URL，浏览器就会使用HTTP向服务器请求相应的资源，如果服务器无法找到资源，就 会向浏览器报告我们熟悉的“404错误”。 绝对路径位于网站名之后，它告诉服务器如何从根文件夹到达资源所在位置。网站名后的第一个“/”代表的就是根文件夹。
1.也许你会问我们在<a>元素的href属性里放入的是相对路径，服务器该怎么根据它找到资源呢？其实浏览器是会根据这个相对路径和这个链接所处页面的路径 创建一个绝对路径的，所以所有Web服务器看到的都是绝对路径，这要归功于浏览器。
2.有一种情况是，如果你没有明确的输入资源所在路径，只是输入一个像这样的目录“http://www.wangchao.com/images”,那么服务器会查找这个目录下的一个 默认文件，通常默认文件被命名为“index.html”或“default.htm”，不要怀疑，第二个文件名的后缀就是那样，用哪个文件名取决于服务器的类型（事实上第二个 文件名的后缀是Microsoft Web服务器的一个怪癖）。所以，在有些情况下，你输入“http://www.wangchao.com”等同于输入“http://www.wangchao.com/ index.html”。
最后,URL并不是只在浏览器地址栏中输入，它还可以在HTML中使用，例如将其放在一个a元素的href属性中就可以链接到Web上的另一个网站，将一个Web上的图片的 URL放在img元素的src属性中就可以在其所在页面显示该图片。
3.还有其它协议，当浏览器从本地计算机读取文件时会使用file协议，它的写法是这样的:“file:///”，所以一个文件URL是这样的:“file:///www.wangchao.c om/index.html”。
4.有时你会见到这样一个URL:“http://www.wangchao.com:8000/index.html”,这与平时所见的URL相比多出了个“:8000”，它是一个放在HTTP URL中的可选 的“端口”，可以这样来考虑它:网站名就像一个地址，则端口就是这个地址的邮箱号，通常Web上的所有东西都会传送到一个默认端口80，只是有时需要在另外一个不同 的端口接受请求比如8000，这种情况经常出现在测试服务器上，正常服务器几乎都在端口80接受请求，如果没有指定端口，则默认为80。

四、两种路径的使用

一般情况下，使用相对路径来链接同一网站下的内容，使用URL来链接Web上的其它网站的资源，这样做是比较好的方法。你也可以让所有链接都统一使用URL，这确实是 可以的，但首先的问题是URL很长，不容易编辑，对页面编写者来说影响页面可读性，另外还有一个问题是，如果一个网站全都使用URL来链接到本地内容，那么移动这个网 站或者改变它的名字时，就必须修改所有的URL来指明新网站的位置——这着实挺麻烦的。而如果使用相对路径，只要不对网站内部文件结构进行更改，就不需要对路径做任 何修改，因为链接的路径都是相对的。
