C 语言最早于 1973 年由 Dennis Ritchie 设计并实现，从那时开始 C 语言便开始从贝尔实验室传播到世界各地，它已成为全球程序员的公共语言，并由此 诞生了两个新的主流语言 C++ 和 Java ———— 它们都是建立在 C 语言的语法和基本结构的基础上的，并且现在的服务器端操作系统 Linux 的内核也是用 C 语言编写的，C 语言从诞生到现在经历过多次的修订，但它仍保持着最初的样子 ———— 一种紧凑、有效的工具。

学习 C 语言的第二本书籍推荐《C 程序设计语言》，书的作者就是 Dennis Ritchie，作为语言的设计者，看他的书很有利于读者把握 C 语言的精髓，不过 不适合拿来入门，对初学编程的人来说难度太高，所以应作为学习 C 语言的第二本书籍。C 语言不是一个大型的语言，不需要一本厚书来讲解，所以这本书关于 语言讲解的有效部分仅仅只有 200 页不到。

C 语言简介

C 语言不专用于某一个特定的领域，它的语言限制很少，通用性强，是一种令人愉快的、具有很强表达能力的通用的语言，适用于编写各种程序，它容易学习，并 且随着使用经验的增加，使用者会越来越感到得心应手。C 语言不受限于任何一种操作系统或机器，由于它非常适合用来编写编译器、操作系统、底层驱动，因此 也被称为“系统编程语言”，但它同样适用于编写不同领域中的大多数程序。C 语言提供了很多数据类型，基本类型包括字符、多种长度的整型、浮点型等，另外还 包括通过指针、数组、结构体、联合体而派生出来的各种自由组合的数据类型。

C 语言的表达式由运算符、操作数组成，任何一个表达式，包括赋值表达式和函数调用表达式，都可以是一个语句，语句用分号结尾。C 语言的基本控制流结构 包括：顺序、条件选择（if）、多路选择（switch）、终止在顶部的循环（while、for）、终止在底部的循环（do-while）、循环跳出（break、continue） 等。

函数可以返回基本、结构、联合、指针类型的值，任何函数都可以递归调用，局部变量默认是“auto”的，即每次调用时重新创建，函数、变量的作用域都是可控的。

编译的预处理阶段将对程序文本进行宏替换、包含其它源文件、条件编译。

C 语言不提供直接处理诸如字符串、集合、列表、数组等复合对象的操作，虽然结构体可以被作为一个单元整个拷贝，但没有处理整个数组或字符串的操作，除了 由函数的局部变量提供的静态定义、堆栈外，C 本身没有定义任何存储器分配工具，也不提供堆、无用内存回收工具，C 本身也没有提供输入/输出（I/O）功能， 也没有内置的文件访问方法。所有这些相对高层的机制都必须由已封装的 C 函数提供。类似的，C 只提供简单的单线程控制流， 即测试、循环、分组、子程序，它不提供多道程序设计、并行操作、同步、协同例程。尽管缺少这些特性好像是个严重不足（这意味着必须通过封装函数来实现这 些复杂功能），但是 C 语言保持在这样的规模是有很多益处的，比如，由于 C 语言相对较小，因此可以用很短的篇幅将它描述出来，容易学会，易于掌握，即程 序员可以很轻松的掌握完整的 C 语言。

C 语言广泛传播的初期，各地出现了很多不同的 C 编译器，这促使美国国家标准协会（ANSI）于 1983 年成立了一个委员会以制订一个统一的、全面的 C 语言 标准，最后结果是 1988 年完成的 ANSI 标准，即“ANSI C”，该标准对 C 进行了一些细微的改进，例如函数的声明定义、允许结构赋值、可进行单精度运算、 明确了算术运算的属性、对预处理器进行详尽的说明等。其次该标准的第二个重要贡献是为 C 定义了一个标准函数库。该库封装了诸如访问操作系统、格式化I/O、 内存分配、字符串处理等类似的一系列函数。该标准还定义了一系列标准头文件，它们为访问函数声明、数据类型声明提供了统一的方法。

C 语言不允许指针和整型数据间交换值，但并没有禁止。C 一直贯彻其最初的设计思想，使程序员了解他们在做什么，对程序员唯一的要求是要明确的表达他们的 意图，C 被公认为一种非常高效的、表示能力很强的语言。

C 语言除法运算法符的特性： #include<stdio.h> int main() { double f; int i; printf("\n 华氏\t\t摄氏"); for (i = 0, f = 0; i < 16; i++, f += 20) printf("\n %g\t\t%g\t\t%g", f, (5.0 / 9) * (f - 32), (f - 32)); //第二个参数若是换成(5 / 9) * (f - 32)，即去掉 5.0 的 .0 ，会发生什么情况？为什么？ printf("\n\n "); return 0; } //也可以借助自动类型转换，可以将式子写成这样 5 * (f - 32) / 9 。 例子修改版，语义化，使程序的目的更明显，更便于阅读。 #include<stdio.h> int main() { double f; int lower = 0, upper = 300, step = 20; printf("\n 华氏\t\t摄氏\n"); for (f = lower; f <= upper; f += step) printf("\n %g\t\t%g\t\t%g", f, 5 * (f - 32) / 9, (f - 32)); printf("\n\n "); return 0; } // lower 代表下限，upper 代表上限，step 代表步长 有利有弊，语义化使程序更容易理解，方便日后维护、扩展，但上例中创建了更多的变量，占用了更多了内存，拖慢了程序的运行速度。
为了改进这一点，可以将这些为了语义化而添加的变量换成宏定义，这样就不会创造出多余的变量，也不会拖慢运行速度。
