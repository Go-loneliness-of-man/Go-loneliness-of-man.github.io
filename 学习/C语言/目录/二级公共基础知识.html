概述

二级 C 概述

全国计算机等级考试简称 NCRE，设四个级别，每年开考两次，一般在三月和九月，实行百分制，同时还划分为四个等级：优秀、良好、及格、不及格，“优秀”等 级的考生合格证书上会注明 “优秀” 字样。

二级 C 考试大纲

公共基础知识 10 分，C 程序设计 90 分，考试时间 120 分钟。
题型是单项选择题 40 分、操作题 60 分（填空 18、修改 18、设计 24）
考试环境：以前是 VC++6、2017 年已换成 VC++2010。

数据结构与算法

算法基本概念 定义：对解决问题的方法的准确而完整的描述。算法不等于程序。
特点：有效性、确定性、有穷性、零个或多个输入、一个或多个输出。
  1. 有效性，算法是有效的，可以解决问题的，能够得到满意结果的。
  2. 确定性，算法的每一步必须有明确的定义，不允许有歧义。
  3. 有穷性，必须能在有限时间内做完。
两个基本要素：对数据的运算和操作、控制结构。
描述算法的工具：程序流程图（PFD）、结构化流程图（N-S）、PDL 等。
时间复杂度：执行算法所需要的计算量，计算量是用所执行基本运算的次数来度量的，同一算法的时间复杂度会随数据的不同 而发生变化。
空间复杂度：指执行算法时所需要的内存空间（算法程序所占空间、输入数据所占空间、算法执行所占空间）。
基本设计方法：枚举、归纳、递推、递归、减半递推、回溯法。
  1. 枚举法，列举所有可能情况并予以处理。
  2. 归纳法，通过分析少量特殊情况，找出一般关系。
  3. 递推法，从已知条件出发，逐步推出中间结果、最终结果。
  4. 递归法，将问题逐层分解进行解决。
  5. 减半递推法，“减半”是指将问题规模减半，而问题性质不变。“递推”是指重复进行“减半”过程。
  6. 回溯法，通过分析问题找出解决问题的思路，沿着思路不断进行试探。
数据结构基本概念 基本概念：数据结构是一门研究数据的组织、存储、操作的学科。
主要目的（宗旨）：提高处理数据的速度、节省处理数据过程中所占用的存储空间。

数据结构一般包含两方面信息，数据信息（数据域）、关系信息（指针域）。

数据元素：数据元素是数据的基本单位，常称节点、记录。
数据项：数据元素由若干数据项组成，数据项是数据元素的最小单位，一般称为成员。
逻辑结构：数据元素间的逻辑关系，其中前后件关系是数据元素间的最基本关系，四种基本逻辑结构是线性结构、树型结构、 网状结构、集合。
  1. 线性结构，元素间是一对一的，像一条线串起来的一样。
  2. 树型结构，元素间是一对多的，像金字塔那样的树形。
  3. 网状结构，元素间是多对多的，节点间随意连线，没有限制。
  4. 集合，元素间的关系是“同属一个集合”，没有连贯所有元素的前后件关系，是种松散的结构。
线性与非线性结构：根据前后件关系的不同，又分为线性结构和非线性结构。线性结构是指 有且只有一个根节点，每个节点最多只有一个前驱，最多只有一个后继的结构。非线性结构指不是线性结构的结构，线性结构 主要有线性表、栈、队列，非线性结构有树、图。
存储结构（物理结构）：逻辑结构在计算机中的存放方式，一种逻辑结构可表示为多种存储结构，程序执行效率与存储结构密 切相关，常用存储结构有顺序存储、链式存储、索引映射、散列映射。
数据的操作：增、删、改、查、插入、排序等。
线性表与顺序存储 定义：由一组相同数据类型的数据元素构成的有限序列。
非空线性表的特征：有且仅有一个根节点，它无前驱，有且仅有一个尾节点，它无后继，其它节点各有一个前驱、后继，节点 个数称为线性表的长度。
顺序存储：在一块连续的存储空间上，将逻辑上相邻的数据元素存储在物理上相邻的存储单元中，该线性表被称为顺序表。
顺序表的特征：1. 所有元素所占存储空间是连续的。2. 各数据元素在存储空间中是按逻辑顺序依次存放的，即逻辑结构与存 储结构一致。
线性表基本操作：增、删、改、查、初始化、求长度、排序。
随机查找：在顺序表中，只要知道首地址，元素序号，便可通过计算找到找到该元素的存储地址，这被称为随机查找。
顺序表插入：将该位置后的元素整体后移一位，时间复杂度 O(n)
顺序表删除：将该位置后的元素整体前移一位，时间复杂度 O(n)
链式存储：将逻辑上相邻的数据元素存储在随机位置的存储单元中，该线性表被称为链式表。
栈和队列 定义：栈、队列是两种使用时有特殊限制的数据结构，所以也称为限定性的数据结构。
栈：只能在线性表的一端（栈顶）进行插入（进栈）、删除（出栈），被称为后进先出的线性表，栈一般有个变量 top 用来 指向当前的栈顶，可能还有个 bottom 指向栈底。使用顺序存储的栈被称为顺序栈。
  1. 当栈的容量超出存储极限时，被称为栈溢出。
  2. 空栈：当 top = 0（实际使用时应该是 -1，但书上说 0，那就 0 吧）或 top = bottom 时，判定为空栈。
  3. 栈中元素个数 = 栈顶 top - 栈底 bottom
队列：只能在一端（队尾）插入（入队），在另一端（队头）删除（出队），被称为先进先出的线性表。队列也有个变量 front 指向队头，同时有个 rear 指向队尾。
  1. 空队列的 rear = front = -1。
  2. 队列的元素个数 = rear - front
  3. 入队时 rear + 1，出队时 front + 1。
  4. 首位相接的队列被称为循环队列，空队、满队时都有 rear = front，但空队时 n = 0，满队时 n = MAXSIZE。
  5. 循环队列中，rear 可以大于 front，front 也可以大于 rear。
链表 链式存储结构：由节点组成，节点又分为数据域、指针域，数据域存放元素本身的数据，指针域存放后续节点的地址。
  1. 链式存储结构的数据密度小于顺序存储（存在指针域）。
  2. 逻辑上相邻的节点，在物理上不相必邻。
  3. 插入、删除等操作时非常灵活，不必移动节点，只需改变指针域。
  4. 既可表示线性结构，也能表示非线性结构。
  5. 采用顺序存取的方式，查找特定元素时必须从开头开始一个个查找。
链表分类：单链表、双向链表、循环链表，从字面意思就很好理解，不细写。
树与二叉树 定义：一种非线性结构，节点间有明显的层次结构关系。
特点：有且只有一个根节点没有双亲，根节点和其它节点有多个互不相交的子树。
节点的度：节点拥有的子树。
节点层次：从根节点算起，根节点为第一层。
叶子：度为 0 的节点，也称端节点。
深度：树的最大层数。
森林：树的集合。
二叉树：每个节点只有两个子树的树，且子树有左、右之分，左、右顺序不能颠倒。
二叉树的五种形态：五种，依次是空二叉树、仅有根节点、右子树空、左子树空、左右均非空。
满二叉树：所有节点都存在左、右子树，所有叶子在同一层的二叉树。
完全二叉树：除最后一层外，其它层都满，并且最后一层的叶子优先集中在左边。所以满二叉树肯定是完全二叉树，反之不成立。
基本性质：
  1. 二叉树第 n 层至多有 2n-1次方个节点。
  2. 深度为 h 的二叉树最多有 2h- 1 个节点。
  3. 在任意二叉树中，假设度为 0 的节点有 x 个，度为 2 的节点有 y 个，则 x = y + 1。
  4. 对于 n 个节点的完全二叉树，其深度为 [ long2n ] + 1，中括号的意思是取整。
二叉树存储结构：顺序存储和链式存储。
顺序存储：若父节点下标为 i，则左子树在 2 * i 处，右子树在 2 * i + 1 处，必须按完全二叉树的形式存储，所以顺序 存储容易有空间浪费。
链式存储：每个节点的指针域含两（有时根据需要还有第三个，用于指向父节点）个指针，分别指向左、右子树。
二叉树遍历：指按某种路线寻访二叉树的每个节点，并且每个节点只访问一次。一般分为先序 遍历、中序遍历、后序遍历，先、中、后指的是根节点在访问中的次序，例如先序遍历指先访问根节点，再左子树，再右子树。无论哪 种遍历，对于子树，都是先左再右。
查找、排序 查找：在给定数据结构中，根据给定条件找到满足条件的点的方法。
平均查找长度：查找过程中平均比较的次数。
顺序查找：用于线性表的查找，从表的一端开始依次比较，直到找到或到达另一端。平均查找长度为 1 / 2，时间复杂度 O（n），一般用于无序或链式存储的线性表。
二分法查找：采用逐步缩小范围的思想，直到找到或确认不存在查找目标为止。前提是数据结构必须是有序的，最坏情况下时 间复杂度为 O（log2n）。
排序：将一个数据元素的无序序列，重新排列为按关键字有序的序列。
排序方法：主要分插入、选择、交换、归并排序，其中插入排序又分直接、折半、希尔排序，选择排序分简单选择、堆排序， 交换排序分冒泡、快速排序。
直接插入排序：从线性表的第二个元素开始，依次取出每个元素，对于每个元素，遍历左方序列（即从第一个元素到当前元素 的序列），将其插入到左方序列的合适位置，直到取到末尾。时间复杂度 O（n2）。
折半插入排序：与直接插入排序基本相同，只是在插入左方序列时，应用二分查找来找到合适的插入位置。
冒泡排序：遍历时比较相邻两元素的大小判断是否交换，这样每次遍历将一个最大/小的值交换到末尾，循环遍历 n - 1 次 即可。时间复杂度 O（n2
简单选择排序：每次遍历从序列中找到最大/小的元素放在序列左侧，循环遍历 n - 1 次即可。时间复杂度 O（n2 ）
图 定义：节点（图中称为顶点）间的连接是任意的。
分类：图分有向图、无向图两种。
有向图：连接具有方向性，可用 <节点 1 名，节点 2 名> 表示从节点 1 到节点 2 的连接，这个连接被称为弧， 节点 1 被称为为弧尾或初始点，节点 2 为弧头或终端点。
无向图：连接不具有方向性，用（节点 1 名，节点 2 名）表示两节点间的一条连接，称为边。
图的表示：用 G = （V，E）来表示图，G 是图的名称，V 是图的顶点数，E 是弧或边的数量。
相关概念：
  1. 权：弧、边的数字，代表其权重。
  2. 网：带权的图。
  3. 顶点的度：某顶点的弧、边数。
  4. 出度、入度：仅针对有向图，分别是以该顶点为尾、头的弧数。
  5. 路径：两顶点间的一条路径，路径上弧、边的数量称为路径的长度。
  6. 连通图：针对无向图，当所有顶点都是连通的，便称连通图，n 个顶点的连通图至少有 n - 1 条边。
本章习题总结
  1. 算法的时间复杂度取决于问题的规模、待处理的数据的初始状态。
  2. 描述算法的常用方法有PFD（程序流程图）、N-S、PDL（伪码）。
  3. 数据结构中，与所使用的计算机无关的是数据的逻辑结构。
  4. 根据数据结构中各数据元素间前后件关系的复杂程度，一般将数据结构分为线性结构、非线性结构。
  5. 数据处理的最小单位是数据项。
  6. 线性表采用链式存储时，则存储单元地址连续不连续都可以。
  7. 在一个长度为 n 的顺序表中，向第 i 个位置插入一个元素时，需要向后移动 n - i + 1 个元素。
  8. 顺序表是随机存取的存储结构，链式表是顺序存取的存储结构。
  9. 在一个长度为 n 的顺序表中，若在任意位置插入一个元素的概率相等，则插入一个元素需移动的元素个数平均为 n / 2。
  10. 栈、队列的共同特点是只允许在端点处插入、删除元素。
  11. 递归中变量的存储分配通常用栈。
  12. 栈通常采用的两种存储方式是链式存储、数组。
  13. 在顺序栈中进行退栈操作时，应先取出元素，后移动栈顶指针。
  14. 循环队列中元素的个数由队头、队尾共同决定。
  15. 链表不具有的特点是可以随机的访问任一元素。
  16. 循环链表的主要优点是从表中任一节点出发，都能访问整个链表。
  17. 树是节点的集合，它的根节点数目是0 或 1。
  18. 在深度为 5 的满二叉树中，叶子的个数是16个。
  19. 在二叉树的三种遍历中，叶子节点的访问顺序完全相同。
  20. 具有 3 个节点的二叉树，有5种可能的形态。
  21. 在深度为 7 的满二叉树中，度为 2 的节点共有63 个。
  22. 假设线性表的长度为 n，则最坏情况下，冒泡排序需要的比较次数为n（n-1）/2。
  23. 对 n 个元素进行直接插入排序，在第 i 趟前，有序部分的元素个数为 i 个。
  24. 对 n 个元素冒泡排序，第一趟至少要进行 n - 1 次比较。
  25. 算法又穷性指的是算法的运行时间是有限的。
  26. 算法的时间复杂度与空间复杂度没有必然联系。
  27. 支持子程序调用的数据结构是栈。
  28. 一个空栈依次入栈 A、B、C、D、E，则它依次出栈的顺序是E、D、C、B、A。

程序设计基础

程序设计方法和风格 程序是指令的集合，执行程序就是解释指令 程序设计语言：计算机本身只能识别二进制代码。其上是中低级的汇编语言，再往上是高级语言，如 C、Jave、PHP 等等。
程序设计风格：指编写程序时所表现出来的特点、习惯、逻辑思路。良好的程序设计风格是程序质量的保证，一般准则是： 清晰第一，效率第二。一般从以下四点规范：
  1. 源文档程序化：标识符命名语义化、多加注释、视觉组织（空格、空行、缩进等）
  2. 数据说明：所有变量先声明再使用，对复杂数据结构应添加注释。
  3. 语句的结构：清晰第一、效率第二。不用或少用 goto 语句，只使用三种基本结构。程序编写模块化。
  4. 输入输出：对所有输入进行合理性检查、对输出格式进行控制
三种程序设计方法：结构化设计（过程化）、面向对象设计、快速原型法。
结构化设计：自顶向下、程序各个部分按功能进行组合。
面向对象设计：以对象为核心，与人类习惯的思维方法一致。
快速原型法：利用现有的工具和原型方法快速的开发所需要的程序。
对于现在的程序设计，一般具体的模块、内部实现采用结构化，而整个程序的架构采用面向对象设计，即程序设计可以是交叉的。
要点整理：
1. 什么是程序？指令的集合。程序设计风格、规范有哪些？
2. 程序设计三种基本方法：结构化设计、面向对象设计、快速原型法。
结构化程序设计 结构化程序设计：
使用三大基本结构，即顺序、选择、循环来设计程序，无论程序多么复杂，都可以利用这三种结构按一定顺序组合起来，从而完成整个程序的设计。
基本原则：自顶向下、逐步求精、模块化、限制使用 goto
要点整理：
1. 结构化设计的四个主要原则：自顶向下、逐步求精、模块化、限制使用 GOTO。
2. 三种基本结构。
面向对象程序设计 基本思想：任何事物都能被看作对象，主张以对象间的关系来如实的反映问题中固有事物间的关系。
与结构化的比较：结构化的分解突出过程，强调代码的功能是如何实现的。面向对象的分解突出现实世界和抽象的对象，突出做什么？
主要优点：
  1. 与人类的思维习惯一致
  2. 稳定性好
  3. 可重用性好
  4. 易于开发大型软件产品
  5. 可维护性好
面向对象设计中主要考虑的是提高软件的可重用性，而实例化、继承性是面向对象实现重用性 的主要机制。
面向对象基本概念：
对象：任何实体、实体抽象、人为概念、有明确边界或意义的东西。对象 = 对象名 + 数据（属性） + 作用于数据的操作 （行为）。基本特点：标识唯一性、分类性、多态性、封装性、良好的模块独立性。
消息：对象之间进行通信的一种数据构造，包括三部分：接受消息的对象名、消息名称、参数（0 或多个）
类：类是具有共同属性、方法的一组对象的集合，是对象之上的抽象，对象是类的具体化，是类的示例。
封装：类封装了同一类对象的属性和方法。封装实现了对内部信息的保护，对外部的信息隐藏， 封装还定义了接口，为外部程序提供调用。
继承：继承是父类和子类间实现共享数据的机制，子类可以继承它祖先类中的属性和操作， 也可以定义自己的属性和操作。一个子类只继承一个父类就叫单重继承，继承多个父类就叫多重继承。
多态性：不同对象收到同一消息产生不同行为的现象叫做多态性，多态的实现受继承支持。
要点整理：
1. 什么是对象？什么是类？
2. 对象的基本组成（对象名、属性、操作）、对象的五个基本特点（标识唯一性、分类性、多态性、封装性、模块独立性）
3. 类的基本特点：抽象性、继承性、封装性、多态性。
本章习题讲解
  1. 不属于结构化程序设计原则的是可封装。
  2. 面向对象具有封装性、继承性、多态性等几大特点。
  3. 面向对象中，继承是指类之间共享属性和操作的机制。
  4. 面向对象中，实现信息隐藏的依靠对象的封装。
  5. 类的示例化不能是一个集合，而是一个具体的对象或单个具体的值，不能是集合。
  6. 面相对象中，有些对象可以不具有继承性。
  7. .面向对象的设计方法与传统的面向过程的方法有本质不同，它的基本原理是使用现实世界的概念抽象地思考问题从而自然地解 决问题。

软件工程基础

基本概念 软件的三部分：程序、相关数据、说明文档
软件按功能分类：系统软件、支撑软件、应用软件
软件的特点：
  1. 抽象性，软件是逻辑实体，不是物理实体
  2. 软件没有明显的制造过程
  3. 软件在使用时，没有磨损、老化问题
  4. 软件、依赖硬件与环境，导致了移植问题
  5. 软件是复杂的，软件的成本非常昂贵
  6. 软件工作涉及到很多社会因素
软件危机：早期软件多是个体制作的，缺少相关文档、质量低、维护困难，这些称为软件危机。软件的 特点导致了软件危机，而软件危机导致了软件工程概念的出现。
软件工程：指应用计算机科学、数学、管理学等原理，以工程化的原则、方法来解决软件问题的工程。其 目的是提高软件生产率、质量，降低软件成本。
基本目标：在给定成本、时间内，开发出具有有效性、可靠性、可理解性、可维护性、可重用性、可适应性、可移植性、可追 踪性、可互操作性且满足用户需求的产品。
原则：抽象性、信息隐蔽、模块化、局部化、确定性、一致性、完备性、可验证性。
抽象性：抓住事物最基本的特性和行为，忽略非本质细节，采用分层次抽象、自顶向下、逐步细化来控制软件开发过程中的复 杂性。
信息隐藏：采用封装技术，将程序模块的实现细节隐藏起来，使模块接口尽量简单。
模块化：模块即程序中相对独立的部分，是独立的编程单位。
局部化：保证模块之间是松耦合的，模块内部是高内聚的。
确定性：软件中所有的概念是确定的、无歧义的、规范的。
一致性：内外部接口保持一致，系统说明与系统行为一致。
完备性：软件系统不丢失任何重要部分，完全实现需求所需的功能。
可验证性：容易被检查、测评、评审。
软件开发技术：包括软件开发方法学、开发过程、软件工具、软件工程环境，其中主要内容是软件开发方法学。
软件开发三要素：方法（技术手段）、工具（开发工具）、过程（开发各个环节的管理理论）
软件开发工具：用来辅助软件开发、运行、维护、管理、支持等活动的软件。
软件开发环境：支持软件开发的软件系统，由软件工具和环境机制构成。
软件生命周期：软件从提出、实现、使用维护、停止使用退役的整个过程。软件生命周期分为三大阶段，定义、开发、维护。
定义阶段：
分问题定义、可行性研究、需求分析三步。
问题定义：确定软件的开发目标、总体要求。
可行性研究：判断软件是否能被实现。确定软件的可行性解决方案、制订开发方案。
需求分析：对软件提出的需求进行分析并给出定义。
开发阶段：
分概要设计、详细设计、实现、测试四步。
概要设计：又称结构设计，讲需求转化为软件体系结构。
详细设计：确定每个模块的实现算法和局部数据结构。
实现：将前两步的设计转化成程序代码。
测试：确定软件测试计划，检验软件各部分是否正确，白盒测试、黑盒测试。
维护阶段：
两步：使用和维护、淘汰。软件投入使用后便一直在第一步中，直到淘汰，进入第二步。
使用和维护：投入运行，不断维护修改，进行必要的扩充、删改。
淘汰：软件通过维护后仍不能满足需要，将被淘汰。
例题：
  1. 软件是指程序、相关数据、文档的集合
  2. 软件生命周期是指软件从提出、实现、使用维护、停止使用退役的过程。
  3. 软件生命周期中不包括市场调研。
  4. 软件设计阶段包括概要设计和详细设计，细分为软件结构设计、数据（数据结构、算法、数据库）设计、接口设计、过程设 计。
  5. 软件生命周期中，详细设计属于开发阶段
  6. 在软件开发中，需求分析阶段产生的主要文档是软件需求规格说明书。
  7. 软件需求规格说明书的作用是：便于用户与开发人员理解交流、反映软件的问题结构，作为软件开发的基础依据、作为测试和验 收的依据。不是可行性研究的依据。
结构化分析方法 结构化分析方法是软件生命周期中需求分析阶段的一种方法。
需求分析：发现需求、求精、建模、定义需求的过程，需求分析将创建所需的数据模型、功能模型、控制模型，是软件生命周 期中最重要的部分。
需求分析的方法：结构化分析方法、面向对象分析方法。
结构化分析方法：目的是弄清用户对软件的需求，建立软件的逻辑模型。结构化分析方法又包含三种，面向数据流（SA）的结 构化分析方法、面向数据结构的系统开发方法（JSD）、面向数据结构的结构化数据系统开发（DSSD）方法。
面向对象分析方法：面向对象软件工程方法在需求分析阶段的应用。
需求分析的四步：需求获取、需求分析、编写需求规格说明书、需求评审。
分析策略：自顶向下、逐层分解。
常用工具：数据流图（DFD）、数据字典(DD)、判定树、判定表
需求分析的结果：软件需求规格说明书（RSR）
软件需求规格说明书：便于用户与开发人员理解交流、反映系统问题结构、作为软件开发的基础和依据、作为软件测试、验收 的依据。其评价标准有正确性、歧义性、完整性、可验证性、一致性、可理解性、可修改性、可追踪性，其中最重要的是正确性。
数据流图（DFD）：用图形描述数据在系统中流动、处理的过程，反映了软件必须完成的逻辑功能，是用于表示系统逻辑模型 的工具。
数据流图的图形：圆形表示加工、转换。箭头表示数据流动方向。两条横线代表文件（数据源）。矩形代表接口（源、潭），即软件、系 统的边界，是系统之外的实体
画数据流图的基本步骤：自外向内、自顶向下、逐层细化、完善求精。
数据流图示例： 数据字典（DD）：与系统相关的数据的定义的集合，对数据流图中被命名的图形元素的确切解释。数据 字典是结构化分析方法的核心
数据字典 5 要素：数据项、结构、流、存储、处理。
判断树：又称决策树，是种描述“加工”的图形化工具，适合描述具有多个判断，每个判断又与多个条件 相关的问题。使用判断树时，要先分清问题中的判断、判定条件、判定条件间的关系（从属、并列、选择），根据这些构造判断树。 判定表：当某个加工依赖多个条件的各种取值，这种情况适合使用判定表。

结构化设计方法 需求分析主要解决了软件“做什么”的问题，确定软件的逻辑模型。而软件设计主要解决“怎么做”的问题，基本目标是用抽象概括的方式 确定软件如何完成预定任务，确定软件的物理模型。
从技术上讲，软件设计分四步：结构设计、数据设计、接口设计、过程设计。
结构设计：定义软件各主要部分间的关系。
数据设计：将概念模型转化为适用于具体 DBMS 的逻辑模型。
接口设计：主要描述软件内部、软件与软件间、软件与人间如何通信。
过程设计：把软件各部分转换为过程性描述。
从工程角度看，软件设计分概要设计、详细设计。
概要设计：又称结构设计，将软件需求转化为软件系统结构，确定系统级接口、全局数据结构或数据库模式、编写概要设计文 档、进行概要设计评审。
详细设计：确定每个模块的实现算法、局部数据结构，用适当方法表示算法和数据结构细节。
软件设计基本原理：抽象化、模块化、信息隐藏、模块独立性。
抽象化：一种思维工具，提取事物的本质特点，忽略细节，抓住本质，降低软件复杂度。
模块化：自顶向下将一个复杂软件划分成较小的、相对独立的模块的过程。使每个功能便于实现。
信息隐藏：模块细节对其它模块来说是隐藏的。
模块独立性：指每个模块只涉及软件具体的子功能，并与其它模块的接口是简单的。模块独立性指标有 耦合性、内聚性。模块划分的原则是低耦合度、高内聚度。
概要设计：
基本任务：设计软件结构、数据结构及数据库设计、编写概要设计文档、概要设计文档评审。
软件结构设计工具——结构图：又称程序结构图，用图形来表示各模块间的关系，如下图： 结构图的几种四种模块：传入、传出、变换、协调模块。
传入模块：从下属处获取数据，处理后再传给上级模块。
传出模块：从上级处获得数据，处理后传给下属模块。
变换模块：从上级处获得数据，处理后再传给上级模块。
协调模块：对所有下属进行协调、管理的模块。 结构图相关术语：
深度：表示整个结构图从属关系的深度。
宽度：模块最多的那一层，就称为宽度。
扇入：该模块直系上级的个数。
扇出：该模块直系下属的个数。
原子模块：没有下属的模块称为原子模块
概要设计通常采用的是面向数据流的设计方法。面向数据流定义了一些映射方法，可以把数据流图转换成结构图表示软件结构。
数据流类型：变换型、事务型。
概要设计基本准则：
  1. 优化软件结构，提高模块独立性。
  2. 模块规模不宜过小，也不能过大。
  3. 模块的作用范围应仅限于模块范围内。
  4. 软件结构的深度、宽度、扇入、扇出要适当。
  5. 要尽量降低模块接口的复杂度
  6. 模块只有一个入 / 出口
  7. 模块输出结果可预测
详细设计：
基本任务：为每个模块确定实现算法、局部数据结构。用特定的表达工具表示算法和数据结构的细节。
常用表达工具：PDL（伪代码）、PFD（流程图）、N - S（方盒图）、PAD（问题分析图）。
流程图中，矩形代表处理步骤、菱形是判断、平行四边形是输入输出。 例题：
  1. 软件设计一般分为两步，是概要设计、详细设计。
  2. 在软件设计中不使用的工具是数据流图（DFD）（结构化需求分析工具）
  3. 数据字典包含的对象都包含于数据流图（DFD）
  4. 结构化程序设计中，模块划分的原则是低耦合、高内聚。
软件测试方法 目的：以最少的人力、时间发现潜在的错误、缺陷。测试用于检查错误，并不修改 错误。
意义：保证软件能够满足需求、可靠性
人员：由开发人员、用户共同完成。
测试基本方法：静态测试、动态测试、
静态测试：又称人工测试。不运行软件，通过评审软件文档，程序、代码检查、静态结构分析、代码质量度量等方法测试，主要通过人工进行。
动态测试：又称机器测试。通过运行软件检验结果来查找错误。主要包括白盒测试、黑盒测试。
白盒测试：又称结构测试、逻辑驱动测试，根据程序的内部结构设计测试例子，对程序的各个路径、过程进行测试。
基本原则：
  1. 保证模块的每个路径至少执行一次。
  2. 保证每一循环都在边界条件、一般条件下至少各执行一次。
  3. 验证所有内部数据结构的有效性。
白盒测试用例的主要设计方法：逻辑覆盖、基本路径测试。
逻辑覆盖：指以程序的逻辑结构为基础的样例设计技术。
基本路径测试：对每一条独立执行路径进行测试。
黑盒测试：不考虑程序内部细节、结构、实现方式，仅测试软件功能是否满足需要，主要用于软件的确认测试。
基本方法：等价类划分法、边界值分析法、错误推测法。
等价类划分法：测试所有可能的输入样例。
边界值分析法：根据输入、输出的边界情况进行样例设计。
错误推测法：依靠人们的经验和直觉进行推测，有针对性的设计样例。
软件测试的四个步骤：单元测试、集成测试、确认测试（验收测试）、系统测试。
单元测试：对软件的最小单位——模块进行测试。通常以白盒测试为主，黑盒测试为辅，主要包括模块接口测试、 局部数据结构测试、错误处理测试、边界测试。
单元测试：将各模块组装后进行测试，发现与接口有关的错误，主要包括软件单元的接口测 试、全局数据结构测试、边界条件和非法输入测试，模块组装方式有非增量组装（一次性组装完毕进行测试）、增量组装（从最基本的几个模块组装起来后逐 渐进行测试、组装）。
确认测试：验证软件功能、性能及其它特性是否满足用户要求，用例设计依据是软件需求规格说明书 ，测试方法主要是黑盒测试。
系统测试：与系统的需求定义进行比较，发现软件与系统定义不符合、矛盾的地方，样例设计依据是 需求分析规格说明书，包括功能、性能、操作、配置、外部接口、安全性测试。
软件调试 基本任务：修正测试时发现的错误，由开发人员进行，各模块由其编写者负责调试。
基本步骤：错误定位、错误纠正、回归测试（避免引入新错误）。
调试方法：主要分静态调试（阅读源码）、动态调试（利用调试工具观察参数等）两大类，以静态调试为主，动态调试为辅， 具体方法有三种，强行排错、回溯法排错、原因排除法（演绎、归纳、二分）。
强行排错：打印所有或特定位置参数、利用自动调试工具来排错
回溯法排错：根据结果分析错误的起因，适用于小一点的程序。
原因排除法：演绎法是根据已有的错误用例，设想、枚举出所有可能出错的原因，再逐个排除确定错误原因。归纳法，分析多 个错误用例间的关系找出错误原因。二分法（打断点），在程序的关键点手动赋给正确的值，运行程序，输出正确则错误原因在前半部 分，否则错误在程序的后半部分。

数据库基础

重难点：数据模型、E-R 图、数据库管理系统、关系运算、数据库设计步骤
基本概念 数据：被存储在介质上用来描述事物的能够被识别的物理符号。
数据有两方面，“值” 和 “型”，值描述事物的具体内容，型描述数据表示的形式，例如日期“2018年 8 月 2 日”也可以用“2018/08/02”来表示。
数据库：指存储在计算机内、有组织、可共享的数据集合，它不仅包括数据本身，还包括数据间的关系。
两大特点：集成、共享。数据库产生的主要目的：共享。
数据库管理系统（DBMS）：一种系统软件，用于科学高效的完成数据库的建立、使用、维护。如 oracle、mysql、mongoDB、Access 等。
功能：数据定义、数据存储和管理、数据操作、数据完整性与安全性、数据库并发控制与故障恢复、数据库运行服务。
组成：数据定义语言（DDL，数据定义、数据存储和管理）、数据操纵语言（DML，数据操作）、数据控制语言（DCL，数据完整性与安全性、数据库并发控制与故 障恢复）、实用程序（数据库运行服务），DDL、DML、DCL 共同组成 SQL 语言。
Sql 语句包括：数据定义语言，负责创建、修改、删除表，索引，视图等对象；数据操作语言，负责数据库中数据的插入、查询、删除等操 作；数据控制语言用来授予、撤销用户权限。
数据库系统（DBS）：引入数据库技术后的计算机系统。
组成：硬件平台、软件平台、数据库集合、数据库管理系统（核心软件）、数据库管理员

数据库应用系统（DBAS）：指系统开发人员利用数据库系统开发的，面向某一类实际应用的软件系统。
基本组成：数据库系统、应用软件、应用界面。

数据库系统的发展：
信息是指被加工为特定形式、可用于指导决策的数据。
数据处理是指对数据进行收集、组织、存储、加工、传播等工作，是将数据转换为信息的过程。
数据是信息的载体和表现形式。信息不随数据形式的变化而变化。信息 = 数据 + 数据处理
数据处理中最重要的问题是数据管理，目前经历过五个阶段，人工管理（1950 前）、文件系统（1950 ~ 1960）、数据库系统阶段（1960 后）、分布式数据库 系统、面向对象数据库系统。趋势是独立性越来越高、使用越来越方便、技术越来越复杂。其中前三个是主要过程，这三个阶段的比较如下：

三个阶段的总结：
人工阶段：数据不保存、不独立、不共享、无结构、没有专门的数据管理软件
文件系统：数据可长期保存，有文件系统这个专门的数据管理软件，但独立性低、共享度差、冗余度大、数据记录内有结构，整体无结构
数据库系统阶段：数据结构化、独立性高、共享性高、冗余度小、易扩展，提供了数据安全性、完整性、并发控制、数据恢复等功能。
数据独立性：分为逻辑独立性、物理独立性。
逻辑独立性是指数据库总体逻辑的改变，如修改数据模式、增加数据类型、改变数据间联系等，不需要修改应用程序。
物理独立性是指存储设备更换、存取方式改变等不影响逻辑结构的改变，不需要修改应用程序。
数据库系统详细介绍：
特点：共享、独立、集成、统一。
  1. 实现数据共享、减少数据冗余。数据与特定应用分离，集中存放，共同使用，保证一致性，但无法避免所有冗余。
  2. 采用数据模型，高度集成。利用模型表现数据间的关系，实现数据结构化，实现有效管理。
  3. 数据独立性高。数据相对应用程序更具有独立性，互相不依赖。
  4. 数据统一管理控制。不同应用程序采用统一的方式访问、控制数据库的数据。具有完整性、安全性、并发性。
数据库系统最根本目的是解决数据共享问题
数据库系统的三级模式：外模式、模式、内模式。
外模式：又称为子模式、用户模式，是模式的子集，是用户的数据结构图，包含允许特定用户使用的那部分数据。
模式：又称逻辑模式、概念模式，描述了数据库中数据的整体逻辑结构、特征。是全体用户公共的数据视图，反映了用户对于现实系统的完整 需求。
内模式：又称物理模式，描述数据在外部存储器上如何组织存储，内模式反映了数据库的存储方式。
模式、外模式不直接存储在计算机中，内模式直接存储在计算机外存。
数据库系统的两级映射：外模式/概念模式的映射保证了数据的逻辑独立性，概念模式/内模式的映射保证了数据的物理独立性。
要点整理：
数据库系统基本概念：数据、数据库、数据库管理系统、数据库系统、数据库应用系统
数据库系统的特征：共享、独立、集成、统一
三级模式与两级映射：外模式、模式、内模式。外模式 / 模式，模式 / 内模式
例题：
一、数据管理技术发展的三个阶段里，数据共享最好的是数据库系统阶段
二、数据库技术的根本目标是要解决数据的共享问题
三、数据库系统的核心是数据库管理系统（DBMS）
四、DB、DBS、DBMS 间的关系是DB < DBMS < DBS
五、负责数据库中“查询”操作的语言是数据库操作语言（DML）
六、数据库设计中反映用户对数据要求的模式是模式（概念模式）
数据模型 概念：数据模型是现实世界的模拟，是对现实世界中的概念进行抽象、处理、表示的工具。数据模型是数据库系统的核心和基 础。
数据模型三要素：
数据结构：描述数据的类型、内容、性质、之间的关系等。是数据模型的基础，描述系统的静态特性。
数据操作：数据库中可执行操作的集合（如增、删、改、查），描述系统的动态特性
数据完整性约束：是一组完整性规则的集合，保证数据有效、正确、相容，其包括实体完整性、参照完整性、用户定义完整性
数据模型分类：三大类
概念数据模型：简称概念模型，是对客观世界事物的结构描述、内在联系刻画，主要有E-R 图（实体关系模型），与具体的 DBMS 或计 算机平台无关
逻辑数据模型：又称数据模型，是种面向数据库系统的模型，着重于数据库系统的实现，主要有层次模型、网状模型、关系模型、面向对 象模型等，确定了不同种类的数据库系统及 DBMS
物理数据模型：又称物理模型，是种面向计算机物理表示的模型，它给出了逻辑数据模型在计算机上的物理实现。
三种数据模型的关系：对现实世界抽象的过程

简单概括就是：数据库设计人员首先对现实世界进行抽象形成概念模型（E-R 图），然后再将概念模型转换成 DBMS 支持的逻辑模型（数据库的逻辑形式）， 最后由 DBMS 将逻辑模型转换为物理模型存储在计算机中（计算机的存储形式）。即前两步由人来完成，最后一步由 DBMS 完成。第一个与 DBMS 无关，后 两个与 DBMS 有关系。
概念模型（E-R 图）：
几个概念：实体、实体属性、实体型、实体集。
实体：客观存在且相互区别的事物。可以是实际的也可以是抽象的。
实体属性：实体的某方面特性。
实体型：某类实体的集合。
实体集：多个同类型的实体构成的集合
实体间联系：
有三种类型：两个、多个实体间的联系和单个实体内部的联系。
主要研究两个实体间的联系，两个实体不代表只有两个，而是说任一情况下只考虑两个实体，包括 1 : 1、1 : n、m : n，如下图：

三种常见的逻辑数据模型：层次数据模型、网状数据模型、关系数据模型
层次数据模型：有且仅有一个节点无前驱（双亲）——根节点。其它节点仅有一个前驱。树形结构，便于描述一对多的情况。（树）
网状数据模型：允许一个以上节点无前驱。一个节点可以有一个以上前驱。（网）
关系数据模型：是当前应用最广泛的数据模型，几乎是现在所有 DBMS 都采用的数据模型。以关系代数为基础，所有数据用二 维表表示，二维表就是关系。
特点：
1. 关系数据模型中，都以二维表来表示数据。
2. 实体或实体间的关系都用二维表来表示。
关系数据模型基本术语：关系、关系模式、元组、属性、域、分量、目、基数、关键字（候选码）、主关键字（主码）、外部 关键字
关系：二维表。
关系模式：对关系的描述，相当于函数，对应着一个关系结构，关系名（关系名1，关系名2 ... 关系名n）
元组：二维表的一行，即一条记录。
属性：二维表中的列。
域：属性的取值种类、范围，如文本、数字等。
分量：元组在属性上的取值。
目：关系中属性的个数。
基数：关系中元组的个数，即二维表中记录的条数
关键字（候选码）：在表中能唯一标识某元组的属性（分量）或属性（分量）集合，一个表中可以有多个关键字。
主关键字（主码）：在表的关键字（候选码）中选出较合适一个来作为表的主关键字，一个表只有一个主关键字。
外部关键字：在本表中不是关键字，但在另一个表中是关键字，那么该属性或属性集合在本表称为外部关键字，外部关键字与另一个表的 关键字可以不同命，只要含义相同即可。
关系基本性质（定理）：
  1. 元组个数有限性
  2. 元组唯一性
  3. 元组分量原子性，即属性是不可再分的数据，不能将一条属性再分为多条属性。
  4. 属性名唯一性
  5. 元组次序无关性
  6. 属性次序无关性
关系中的数据约束：保证数据的完整性、一致性、相容性
实体完整性约束：关系的主键不能为空（null）。
参照完整性约束：关系间相互关联的约束，外键要么是实际存在的元组，要么是空值。
用户自定义完整性约束：自定义的数据必须满足的某一要求，例如成绩在 60 以上。
任何 RDBMS（关系数库管理系统）都必须支持实体、参照完整性约束，这称为关系的两个不变性。
要点整理：
数据模型：三要素（数据结构、数据操作、数据约束），三类模型（概念模型、逻辑模型、物理模型）
逻辑数据模型的三种类型：层次、网状、关系模型。
E-R 模型：矩形（实体）、菱形（联系）、椭圆（属性）。
数据约束三种类型：完整、参照、用户自定义完整性约束。

关系代数运算 概念：当要从关系数据库中找到需要的数据时，常常需要对多个关系进行运算。关系运算是以关系代数为基础的。
关系运算基本分为两类：传统集合运算、专门关系运算
传统集合运算：并、交、差、笛卡尔积、除
专门关系运算：选择、投影、连接
并：并、交、差这三种运算都要求关系间有相同的结构。
交：找出两关系中相同的部分。
差：即相减，前者减去与后者相同的部分，剩下的就是差。用 关系1符号 —— 关系2符号 这种形式表示，代表属于关系1但 不属于关系2的元组。
笛卡尔积：设有 n 元（组）关系 R 与 m 元关系 S，则二者笛卡儿积就是个 n x m 元的新关系。通俗讲，就是利用一 个关系的每个元组与另一关系的所有元组都组合一遍，因此才产生了个 n x m 的新关系。
除：除法运算考的很少，考出来也能用排除法做，其计算过程比较复杂。简单说：设有关系 R（X，Y）、S（Y，Z），其中 X 、Y、Z 代表属性组，R 中的 Y 可以与 S 中 Y 名称不同，含义相同即可。R 除 S：
1. 先求 R 内 X 各属性值的象集。
2. 再求 S 在 Y 下的投影。
3. 最后，选择象集能够包含整个投影的属性值组成新关系。
象集是指，X 某属性值对应的所有 Y 内属性值对的集合。
选择：从一个关系中选择满足特定条件的元组（记录），属于横向操作。符号是西格玛，条件（西格玛的右下角）（关系符号）
投影：从关系中选择由若干属性列组成的新关系，符号类似于选择，换成圆周率符号即可，是纵向操作。
连接：从两个关系的笛卡儿积中选择满足一定条件的元组，可以进行多个关系间的运算。分三种形式：条件连接、等值连接、 自然连接
条件连接：从两个关系的笛卡儿积中选择属性值满足一定条件的元组，相当于积 + 选择。
等值连接：当条件连接取 “ = ” 时，就是等值连接。
自然连接：作笛卡儿积，去掉结果中重复的属性列，若两个属性列的属性值不完全相等，只保留相等的部分。
数据库设计方法及步骤 数据库设计是数据库应用的核心，数据库设计是指在已有的 DBMS 上建立数据库。
基本任务：解决用户的信息需求、处理需求，根据 DBMS 设计出数据模式。
根本目标：为用户、各种应用系统提供一个信息基础设施和高效的运行环境，即提供一个满足用户、应用程序需求的数据库系统。
两种基本设计方法：面向数据、面向过程。前者以信息需求为主，后者以处理需求为主。
数据库设计的六个阶段：
  1. 需求分析阶段：了解用户需求，工具是数据流图、数据字典，结果是需求说明书。
  2. 概念设计阶段：结果是概念数据模型（实体关系图，即 E - R 图）
  3. 逻辑设计阶段：结果是逻辑数据模型（根据使用的 DBMS 设计）
  4. 物理设计阶段：结果是数据库，存储在物理结构上的。
  5. 应用程序设计：应用程序的编写与测试
  6. 运行和维护：重组（重新分配数据）、重构（重新设计设计数据模式）、故障恢复、性能监视等
